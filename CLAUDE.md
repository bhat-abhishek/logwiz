# LogWiz ‚Äî Codebase Guide

**LogWiz** is a lightweight TypeScript browser logging library inspired by Winston. It supports multiple log levels, pluggable transports (Console, CloudWatch), automatic browser/OS/screen metadata capture, and log batching.

## Stack

TypeScript ¬∑ Node.js (build-time only) ¬∑ `@aws-sdk/client-cloudwatch-logs` ¬∑ `yarn`

## Commands

```bash
yarn install          # install dependencies
yarn build            # compile TypeScript ‚Üí dist/
```

There is no test runner configured yet. Add one under `scripts.test` in `package.json` when tests are introduced.

## Architecture

### Logger (`src/logger.ts`)

The core `Logger` class batches log entries and ships them to all configured transports once the batch reaches `batchSize` (default: 5). Every entry automatically includes browser, OS, and screen metadata captured at construction time.

```
Logger
  ‚îú‚îÄ‚îÄ log(level, message, meta?)   ‚Äî level-gated; formats and batches the entry
  ‚îú‚îÄ‚îÄ info / warn / error / debug  ‚Äî convenience shorthands
  ‚îú‚îÄ‚îÄ batchLogs()                  ‚Äî accumulates entries; triggers shipLogs when batchSize hit
  ‚îî‚îÄ‚îÄ shipLogs()                   ‚Äî calls transport.log(batch) on every registered transport
```

Log level order (ascending): `debug` ‚Üí `info` ‚Üí `warn` ‚Üí `error`. Entries below the configured level are silently dropped.

### Transport (`src/transports/transport.ts`)

Abstract base class. Subclasses implement:

```typescript
abstract log(logs: LogEntry[], callback?: () => void): void;
```

### ConsoleTransport (`src/transports/console/index.ts`)

Routes each `LogEntry` to the matching `console.*` method (`console.error`, `console.warn`, `console.info`, `console.debug`).

### CloudWatchTransport (`src/transports/cloudwatch/index.ts`)

Scaffold for AWS CloudWatch Logs integration; `log()` body is a stub pending implementation.

### Types (`src/types/loggerTypes.ts`)

| Type | Purpose |
|---|---|
| `LogLevel` | `"info" \| "warn" \| "error" \| "debug"` |
| `LoggerOptions` | Constructor options: `level`, `transports[]`, optional `config` |
| `LoggerConfig` | `batchSize` (count) and `batchTime` (seconds, unused in current impl) |
| `TransportConfig` | Per-transport options plus optional `exceptionHandler`/`rejectionHandler` |
| `LogEntry` | Single log record: `level`, `message`, `timestamp`, optional `stack` + `meta` |

## Conventions

- Add new transports under `src/transports/<name>/index.ts` extending `Transport`.
- Export new transports from `src/transports/index.ts` and `src/index.ts`.
- Keep `dist/` out of version control ‚Äî it is generated by `yarn build`.
- The library targets browser environments; `navigator`, `window`, and `document` APIs are used directly in `logger.ts`.
- Build output is `CommonJS` (`"module": "commonjs"` in `tsconfig.json`); type declarations are emitted alongside JS.
